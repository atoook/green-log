# Product Overview

Green Mate は「植物との生活記録」をテーマにした、観葉植物初心者向けのお世話・成長記録アプリである。単なる作業管理ではなく、植物を枯らさず、成長を感じ、愛着を持って育て続けられる体験を目指す。

管理単位は植物種ではなく、ユーザーが所有している鉢・個体である。同じ種類の植物でも、別の鉢や別の個体であれば別の記録として扱う。植物記録はアカウントに閉じた私的な生活記録であり、ユーザーは自分の植物だけを登録、閲覧、更新できる前提にする。

## Core Capabilities

- 植物個体の基本情報を登録し、一覧と詳細で見返せる。
- 水やり周期やお世話履歴をもとに、今日必要なお世話へつなげる。
- 写真や日付付きの記録を通じて、成長実感と振り返りを支える。
- 初心者が迷わず続けられるよう、空状態・失敗時・未入力時にも次の行動を示す。

## Product Boundaries

Green Mate は植物辞典や育成ガイドを主目的にしない。植物種マスタ、専門的な育成レコメンド、画像ストレージ、共有などは、必要性が明確になった spec で扱う。

認証・認可は共通基盤として導入済みである。後続の水やり記録、今日のお世話、成長写真ログなどのユーザー所有データは、アカウント単位の private data として扱い、所有者分離を前提に requirements と design を作る。ロール、管理者権限、組織、共有、退会後の詳細なデータ保持ポリシーは、別 spec が承認されるまで追加しない。

機能固有の登録項目、API contract、画面遷移、詳細な受け入れ条件は `.kiro/specs/` に残す。steering には長期的に変わりにくいプロダクト方針だけを記載する。

## UX Principles

- ユーザー向け文言は、業務的な「管理」「タスク」よりも「記録」「お世話」「暮らし」の語感を優先する。
- 初心者向けに、操作の選択肢を絞り、今できることと次の行動を明確にする。
- モバイルでも主要情報と主要操作が読み取りやすい layout を優先する。
- 植物との生活を続ける感覚を損なう、過度に事務的な UI や専門用語を避ける。
- 認証状態の確認中、未ログイン、セッション切れでは、保護された植物記録を表示せず、ログインや再認証へ自然につなげる。
